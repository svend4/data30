# Тест-план

## Цели
- Подтвердить готовность ключевых пользовательских сценариев.
- Выявить дефекты, влияющие на выпуск.

## Область тестирования
- Базовые пользовательские сценарии (MVP).
- Ошибочные/граничные условия.

## Предпосылки
- Доступны требования и критерии приемки.
- Развернута тестовая среда и предоставлены тестовые данные.

## Риски
- Отсутствие описанных требований/документации.
- Недоступность окружения или данных.

## Ключевые сценарии
1. Запуск и доступность приложения.
2. Основной пользовательский поток (happy path).
3. Обработка ошибок ввода/некорректных данных.
4. Сохранение/обновление данных.
5. Поведение при недоступности внешних зависимостей.

## Чек-листы
### Чек-лист 1: Доступность и запуск
- [ ] Приложение запускается без ошибок.
- [ ] Главная страница/экран открывается.
- [ ] Основные ресурсы загружаются без ошибок.

### Чек-лист 2: Основной пользовательский поток
- [ ] Пользователь проходит ключевой сценарий от начала до конца.
- [ ] Данные сохраняются корректно.
- [ ] UI показывает ожидаемые состояния.

### Чек-лист 3: Ошибки ввода и граничные случаи
- [ ] Обязательные поля валидируются.
- [ ] Пустые значения обрабатываются корректно.
- [ ] Некорректные значения приводят к понятным сообщениям.

### Чек-лист 4: Обновление и сохранение данных
- [ ] Создание записи работает.
- [ ] Редактирование записи работает.
- [ ] Удаление записи работает (если предусмотрено).

### Чек-лист 5: Внешние зависимости
- [ ] При недоступности зависимостей отображается корректная ошибка.
- [ ] Повторная попытка после восстановления работает.

## Критерии завершения
- Все критические сценарии пройдены.
- Критических дефектов нет (Severity 1).
- Известные дефекты задокументированы и приняты.
