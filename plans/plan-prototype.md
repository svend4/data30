# План прототипирования

## 1. Ключевые экраны
- **Вход**: поля логина/пароля, восстановление доступа, CTA входа.
- **Каталог функций/сценариев**: список карточек, фильтры/поиск, навигация по категориям.
- **Рекомендация**: краткое описание рекомендованного сценария, ожидаемый эффект, CTA «Применить».
- **Подтверждение**: сводка выбора, детали запуска, финальное подтверждение.

## 2. Прототип
- **Формат**: статический HTML (с возможностью переноса в Figma/Storybook при необходимости).
- **Состав**: 4 страницы для каждого ключевого экрана.
- **Навигация**: простые ссылки между экранами (вход → каталог → рекомендация → подтверждение).

## 3. Внутренний обзор и фиксация правок
- **Участники**: продукт, дизайн, разработка.
- **Критерии**: ясность навигации, понятность контента, отсутствие «тупиков».
- **Артефакт**: список правок с приоритетами (P0/P1/P2) и владельцами.
- **Результат**: обновлённый прототип и краткий лог изменений.
