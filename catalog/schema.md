# Единый формат карточки приложения

Каждая карточка описывает приложение в каталоге и хранится в `apps.yaml`.

## Поля

- `id` — уникальный идентификатор (slug).
- `name` — название приложения.
- `platforms` — список поддерживаемых платформ (web, ios, android, desktop, cli, api).
- `functions` — ключевые функции (3–7 пунктов).
- `constraints` — ограничения и лимиты использования (список).
- `price` — ориентир стоимости (free, trial, paid, tiered, custom).
- `documentation` — ссылки на официальную документацию (список).
- `compatibility` — список `id` приложений, с которыми есть проверенная интеграция.
- `support_status` — статус поддержки (active, limited, deprecated, discontinued).
- `maturity` — зрелость (Prototype, Stable, Recommended).
- `use_cases` — сценарии использования (use case) в виде списка.
- `licensing` — сведения о лицензировании и стоимости:
  - `model` — модель лицензирования (open_source, subscription, perpetual, usage_based, freemium).
  - `license` — тип лицензии (например, MIT, Apache-2.0, commercial).
  - `cost` — ориентир стоимости (free, trial, paid, tiered, custom).
  - `notes` — уточнения по условиям.

## Минимальные требования

- Каждая карточка должна содержать все поля.
- `compatibility` не должна содержать собственный `id`.
- Для MVP каталог содержит 20–50 приложений.

## Правила зрелости и назначения

### Уровни зрелости
- `Prototype` — экспериментальный уровень, допускаются ограничения и нестабильность.
- `Stable` — подтвержденная работоспособность ключевых сценариев и приемлемая стабильность.
- `Recommended` — высокая надежность, подтвержденная полезность и наличие поддержки/документации.

### Переходы между уровнями
- `Prototype` → `Stable`: достаточный объем подтвержденных кейсов, стабильность выше порога, нет критических инцидентов.
- `Stable` → `Recommended`: высокая оценка качества и полезности, документированная поддержка и стабильный SLA.

### Импорт и администрирование
- При импорте карточек `maturity` может назначаться автоматически по правилам (если доступны метрики), либо вручную.
- В админ‑панели предусмотрены: ручной выбор уровня, отображение критериев и аудит изменения уровня.
