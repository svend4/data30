# Roadmap релизов: роли, сценарии, ограничения и epic/user stories

## 1. Описание продукта

**Data30** — метаслой и каталог решений/макросов для поиска, повторного использования и оценки готовых сценариев автоматизации и интеграций.

### Целевые роли
- **Product Manager / Owner** — формирует каталог решений, управляет качеством и приоритетами.
- **Инженер / Интегратор** — ищет готовые функции/макросы и переиспользует их в проектах.
- **Аналитик / No‑code пользователь** — выбирает сценарии и запускает макросы без глубокого кода.
- **QA / Compliance** — проверяет соответствие требованиям и трассируемость.
- **Администратор платформы** — управляет доступами, модерацией, стабильностью.

### 3–5 основных пользовательских сценариев
1. **Поиск готового решения**: пользователь находит макрос/сценарий в каталоге по задаче или тегам.
2. **Переиспользование макроса**: инженер копирует шаблон и настраивает параметры под проект.
3. **Оценка и отзыв**: пользователь оставляет отзыв (качество/полезность) с подтвержденным использованием.
4. **Модерация и контроль качества**: администратор проверяет отзывы и снимает спам.
5. **Мониторинг и рекомендации**: команда анализирует метрики и использует рекомендации по сценариям.

### Ключевые ограничения
- MVP должен быть реализован в рамках текущего квартала и фиксированного бюджета.
- Ограниченные инженерные ресурсы и параллельные инициативы.
- Соблюдение требований по приватности данных и хранению отзывов.
- Для рейтингов нужны подтвержденные данные использования.

### Метрики успеха
- Доля требований, покрытых метриками: 100% для MVP.
- Доля подтвержденных отзывов среди всех отзывов.
- Среднее время нахождения подходящего макроса (поиск → выбор).
- Количество повторных использований макросов/сценариев.
- Доля решений, перешедших в статус «рекомендовано».

## 2. Релизы: цели и границы

### MVP
**Цели:**
- Запустить минимальный каталог решений и базовую модель отзывов.
- Обеспечить поиск, просмотр карточки и базовый трекинг использования.

**Границы функциональности:**
- Только базовые поля решений/макросов.
- Простая форма отзывов (качество/полезность).
- Минимальная модерация (ручная проверка, без сложной анти‑спам логики).
- Нет сложной аналитики и продвинутых рекомендаций.

### Beta
**Цели:**
- Улучшить качество данных, ввести подтверждение использования и расширить каталог.
- Запустить ранжирование по сценариям и базовую аналитику.

**Границы функциональности:**
- Частичная автоматизация модерации.
- Базовые рекомендации на основе сценариев.
- Ограниченный набор интеграций для подтверждения использования.

### Scale
**Цели:**
- Масштабирование каталога, продвинутые рекомендации и автоматизация модерации.
- Устойчивое качество и SLA/наблюдаемость.

**Границы функциональности:**
- Полноценная анти‑спам система и доверительный рейтинг.
- Продвинутая аналитика, сегментация и кастомные отчеты.
- Интеграции с несколькими источниками данных и расширенный API.
