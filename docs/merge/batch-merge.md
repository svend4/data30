# Массовое слияние веток

Этот документ описывает базовый порядок объединения большого числа веток
с минимизацией конфликтов в README и планах.

## 1) Рекомендуемый порядок

1. Включить git rerere для запоминания решений конфликтов.
2. Сливать ветки по одной, начиная с тех, где меньше изменений.
3. Использовать автоматическое разрешение README через скрипт.

## 2) Включение rerere

```bash
git config rerere.enabled true
git config rerere.autoupdate true
```

## 3) Скрипт для слияния веток

Используйте:

```bash
scripts/merge-branch.sh <branch>
```

Скрипт:
- подтягивает `origin/main`;
- сливает базовую ветку;
- при конфликте README применяет `scripts/resolve-readme-conflicts.sh`;
- сообщает о оставшихся конфликтах.

## 4) Если остаются конфликты

- README уже будет сохранен в `readmes/`, а основной README восстановлен.
- Конфликты в планах переносите в `plans/plan-roadmap.md`.
- После ручного исправления выполните:

```bash
git add <files>
git commit -m "Resolve merge conflicts"
```
