# Привязка каталога к Function Dictionary

Этот документ описывает, как связывать карточки приложений из `catalog/apps.yaml`
с функциями из `docs/function-dictionary.md` для единых сценариев и макросов.

## 1) Принципы сопоставления

1. Привязывайте 3–7 ключевых функций приложения к словарю функций.
2. Используйте один стиль именования: глагол + объект (например, SendEmail).
3. Если функция приложения шире атомарной, разнесите ее на 2–3 функции словаря.
4. Не дублируйте функции: используйте существующие ID и описания.

## 2) Формат сопоставления

Рекомендуемый формат — таблица:

| App ID | Название приложения | Функции (ID) | Комментарий |
| --- | --- | --- | --- |
| slack | Slack | F037, F039, F040 | Коммуникации и уведомления |
| jira | Jira Software | F050, F051, F053 | Управление задачами и событиями |
| google-drive | Google Drive | F026, F028, F029 | Хранение и работа с файлами |

## 3) Проверка качества

Перед добавлением сопоставления:
1. Убедитесь, что функции существуют в `docs/function-dictionary.md`.
2. Проверьте, что функции соответствуют описанию карточки приложения.
3. При необходимости добавьте уточняющий комментарий.
