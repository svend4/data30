# Обновление карты совместимости

Документ описывает, как поддерживать `catalog/compatibility.md` в актуальном состоянии
после изменений в `catalog/apps.yaml`.

## 1) Когда обновлять

- При добавлении нового приложения в `catalog/apps.yaml`.
- При изменении поля `compatibility` у существующего приложения.
- При удалении приложений или переименовании `id`.

## 2) Правила обновления

1. Каждая связь в `compatibility` должна отражаться в `catalog/compatibility.md`.
2. Используйте те же `id`, что и в `catalog/apps.yaml`.
3. Удаляйте устаревшие интеграции при удалении карточки приложения.

## 3) Быстрая проверка

- Сверьте количество строк в `catalog/compatibility.md` с количеством приложений.
- Проверьте, что нет ссылок на несуществующие `id`.
- Убедитесь, что список совместимости не включает собственный `id`.
